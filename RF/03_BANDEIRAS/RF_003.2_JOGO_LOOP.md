# RF_003.2 - Execução da Partida de Bandeiras

## VISÃO GERAL

Esta funcionalidade descreve a interface e a lógica de interação durante uma partida ativa de reconhecimento de bandeiras. Ela adapta seu layout dinamicamente com base no modo selecionado na **RF_003.1** (Múltipla Escolha ou Flashcards), gerenciando o progresso do usuário até o final da rodada.

## ÉPICO

Como jogador, quero uma interface limpa e responsiva durante a partida para que eu possa focar no reconhecimento das bandeiras sem distrações.

## HISTÓRIA DE USUÁRIO

Como estudante de geografia, quero receber feedback visual imediato ao acertar ou errar uma questão, e visualizar meu progresso em tempo real durante a rodada, garantindo que eu saiba quantas questões faltam para concluir o desafio.

## DETALHAMENTO DE CAMPOS E ELEMENTOS

Esta seção detalha os componentes da interface de jogo, que variam conforme o modo escolhido.

| Campo / Elemento | Tipo | Descrição / Comportamento esperado |
| :--- | :--- | :--- |
| **Painel de Status (Topo)** | Container | Exibe o contador de progresso (ex: "5 / 20"), a pontuação atual e um cronômetro regressivo ou crescente por questão. |
| **Botão: Pausar/Sair** | Botão (Ícone) | Localizado no canto superior. Abre menu de confirmação para interromper a partida. |
| **Área do Enunciado** | Texto ou Imagem | Exibe o "Pergunta" da rodada. Pode ser o Nome do País ou a Imagem da Bandeira, dependendo do modo. |
| **Grid de Opções (Múltipla Escolha)** | Layout de Grid | Renderiza 6 botões (com nomes ou imagens). Ao clicar, o sistema valida a resposta. |
| **Feedback Visual de Resposta** | Animação/Cor | **Acerto:** O botão selecionado fica Verde e o sistema avança. **Erro:** O botão selecionado fica Vermelho e o botão correto é brevemente destacado em Verde. |
| **Interface Flashcard** | Container Tátil | Área central que exibe o item inicial. Ao tocar, realiza uma transição (flip ou revelação) para mostrar o verso com a informação correspondente. |
| **Botões Flashcard (Próximo/Anterior)** | Botão | Navegação manual específica para o modo Flashcard para avançar ou retroceder entre os itens. |

## PRÉ-REQUISITOS

1. Uma sub-região e um modo de jogo devem ter sido validados na **RF_003.1**.
2. O banco de dados de países/bandeiras deve estar carregado no escopo da sessão.

## DEPENDÊNCIAS

- RF_003.1 - Escolha de Bandeiras (para definição do modo e escopo).
- RF_003.4 - Tela de Resultados (para onde o usuário é enviado ao terminar).

## FORA DE ESCOPO

- Compra de itens ou power-ups durante a partida.
- Ranking global em tempo real (competitivo online).

## COMPORTAMENTO ESPERADO

### Cenário 1: Fluxo de Múltipla Escolha (Sucesso)

**Dado que** o usuário iniciou o modo "Nome - 6 Bandeiras",
**Quando** o usuário clicar na bandeira correta correspondente ao nome exibido,
**Então** o sistema deve aplicar a cor verde ao botão, somar pontos ao score e realizar a transição automática para a próxima pergunta após 500ms.

### Cenário 2: Fluxo de Múltipla Escolha (Erro)

**Dado que** o usuário está em uma partida,
**Quando** tocar em uma opção incorreta,
**Então** o sistema deve pintar o botão de vermelho, destacar a resposta certa em verde, e aguardar um breve intervalo ou toque na tela para avançar, sem somar pontos.

### Cenário 3: Interação com Flashcards

**Dado que** o usuário está no modo "Bandeira - Flashcard",
**Quando** visualizar a imagem da bandeira e tocar na tela,
**Então** o sistema deve realizar uma animação de revelação exibindo o nome do país correspondente.

### Cenário 4: Conclusão da Rodada

**Dado que** o usuário respondeu à última pergunta da lista (ex: 20/20),
**Quando** o feedback da última resposta for concluído,
**Então** o sistema deve redirecionar automaticamente o usuário para a **RF_003.4 (Tela de Resultados)**.
